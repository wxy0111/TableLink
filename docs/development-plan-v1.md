# 第一版开发计划

## 阶段 1：项目基础

- 初始化 Next.js 前端
- 初始化 NestJS 后端
- 接入 PostgreSQL 和 Prisma
- 配置环境变量
- 准备 Docker Compose
- 增加基础 seed 数据

## 阶段 2：菜单和桌台

- 后台分类 CRUD
- 后台菜品 CRUD
- 菜品上下架、售罄
- 桌台 CRUD
- 桌台二维码 code 生成

验收：

- 老板能维护菜单。
- 老板能创建桌台并得到扫码入口。

## 阶段 3：顾客点餐

- 通过桌台 code 进入菜单
- 分类和菜品展示
- 规格/口味选择
- 购物车
- 提交订单
- 查看订单状态

验收：

- 顾客能完成扫码、点菜、下单。
- 订单金额由服务端计算。

## 阶段 4：厨房屏

- 厨房订单列表
- 按档口过滤
- WebSocket 实时推送
- 菜品状态更新

验收：

- 顾客下单后厨房屏马上出现新订单。
- 厨房能标记接单、制作中、已完成。

## 阶段 5：服务员和收银

- 桌台状态列表
- 桌台当前订单
- 开台
- 换桌
- 并桌
- 清台
- 服务员加菜
- 催菜
- 等叫
- 取消或退菜
- 创建支付记录
- 微信/支付宝/现金支付方式记录
- 组合支付
- 退款记录
- 标记订单已支付
- 操作流水
- 打印任务基础模型

验收：

- 服务员能处理一桌从开台到结账的完整流程。
- 退菜和支付都有流水。

## 阶段 6：本地部署和局域网访问

- 本地 PC 启动脚本
- 显示本机局域网 IP
- 桌台二维码生成
- 防火墙/端口说明
- 数据库备份和恢复说明

验收：

- 顾客手机可通过局域网 IP 访问桌台菜单。
- 后厨和收银平板可访问厨房屏和收银台。
- 店里无公网时仍可点餐、出餐、结账。

## 阶段 7：统计和上线

- 员工登录
- 角色权限
- 今日订单数
- 今日营业额
- 日结对账
- 操作流水查看
- 打印任务异常查看
- 热销菜品
- 未支付订单
- 支付方式统计
- 日/周/月/季度/年报表
- Docker Compose 部署
- 基础操作日志

验收：

- 小店能试营业使用。
- 老板能看今日、周、月、季度、年经营概况。

## 暂不做

- 智能总结
- 库存
- 会员
- 外卖
- 多门店
- 复杂财务
